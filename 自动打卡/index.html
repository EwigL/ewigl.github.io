记一次自动打卡程序编写过程

发表于 2021-02-08 分类于 笔记

抓包

  • 使用BurpSuite进行抓包，打卡程序客户端为浏览器因此抓包很简单顺利。使用Filter过滤掉无用数据。也可在Options中设置Intercept Responce以拦截Response拦截一些无用Response

  • 对抓取到的数据进行分析，经过分析发现，提交表单使用常用的POST请求。

  • RequestBody 是一个json表单，在程序中使用JsonObject构建即可。表单中包含的时间字段使用Date或者Calendar类获取并Format，某一字段需要使用随机数填充。其余字段均为固定值。

  • RequestHeader中的Cookie字段包含了四个字段，其中两个字段已加密，明显为用户用户名密码加密后的密文。但是似乎并不需要知晓加密方法。因为通过分析前面的HTTP请求部分可知此字段可通过POST请求从服务器获取。

  • 至此，结合前文分析，抓包部分结束。

    • 首先通过站内GET请求获取route字段。
    • RequestHeader携带route，RequestBody携带用户信息，发送POST请求获取字段2、3以及JSESSIONID。
    • Cookie携带route、字段2、3以及JSESSIONID提交表单。

程序

  • 程序主要包含Start入口，ConfigData储存数据，HttpUtil网络相关方法，Login登录，Submit提交表单，SendMail发送程序报告至邮箱。
  • Start入口，主函数中调用登录以及提交表单方法。
  • Login登录,部分代码
    // GetAccess方法返回Cookie // 验证登录 // 获取route 登陆成功后获取到Cookie即可进行表单提交操作。
    // 构建表单 // 提交表单并获取ResponseBody // 获取结果后发送结果至指定E-Mail，MailTitle与MailContent根据程序运行过程逐步添加/更改。

    部署

  • 使用IDEA将程序打包成jar
  • 上传至GitHub
  • 设置GitHub Actions 定时运行程序
  • 注意设置运行环境与IDEA build环境相同
  • 当然也可以上传至私人服务器设置定时任务
  • 由于GitHub Actions 服务器默认时区为GMT，因此在程序起始需要设置时区为GMT+8
